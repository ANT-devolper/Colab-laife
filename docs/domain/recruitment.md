# Recruitment

> **Status:** 🚧 planned — not implemented yet.

## Purpose

Manage hiring: job openings and the candidate pipeline from application to decision.

## Key concepts / entities

- **Opening / position** — a role being hired for.
- **Candidate / application** — a person applying, moving through pipeline **stages**.

## Main flows

- Open a position.
- Receive/register candidates and move them through stages.
- Record a decision (hire / reject), optionally feeding the Employees module.

## Permissions / roles

- Recruiters/admins manage openings and candidates; scoping defined at design time.
