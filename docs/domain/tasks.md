# Tasks

> **Status:** 🚧 planned — not implemented yet.

## Purpose

Assign and track tasks for employees within an organization.

## Key concepts / entities

- **Task** — a unit of work with a title/description, an assignee, a status and (likely) a
  due date.

## Main flows

- Create a task and assign it to an employee.
- Update status / complete a task.
- List tasks by assignee or status.

## Permissions / roles

- Assigners (managers/admins) and assignees; exact rules defined at design time.
